# Session Playbook · 你看的

> 这份是给你自己看的，不是给 AI 的。开 session 前扫一眼，焦虑时拿出来看，翻车后对照。
> AI 协作规则在 [CLAUDE.md](CLAUDE.md)，接口和架构在 [PROJECT_SPEC.md](PROJECT_SPEC.md)。这份只讲**节奏**。

---

## 一句话

**一个 session = 一个切片 = 一个 commit ≤ 90 分钟。**

做不到就是切片拆太大了，不是你水平不够。

---

## 开场三步（每次都做，不许跳）

| 步骤 | 动作 | 通过标准 |
|---|---|---|
| 1 | 让 AI 读 CLAUDE.md + PROJECT_SPEC.md，用一句话复述本次切片目标 + 验收 | 复述准确，不准就你纠正 |
| 2 | AI 先出方案不写代码：改哪些文件、为什么、对应 SPEC 哪一节、要不要加新库 | 你看懂并同意 |
| 3 | 在脑子里/便签上写下「这次结束时我要演示什么动作」 | 一句话能说清 |

> 三步任何一步卡住，宁可花 20 分钟拉直，也不要硬开工。

---

## 中场守则（写代码时）

- **小 diff，每个都过一眼**。看不懂 → 当场停。
- **系统层先看 Handy**：让 AI 把 Handy 对应实现的关键代码贴给你看，再写。
- **AI 要加抽象/库/manager** → 按 SPEC §〇.4 停手。SPEC 没写的，AI 不能自己决定。
- **每跑通一小段就 commit**，WIP commit 也行。git 就是 undo。

---

## 收尾四步（少一步都算没做完）

1. **真演示一遍**：按开场约定的动作跑一次。不是 AI 说"跑通了"，是你眼睛看到。
2. **commit 干净**：WIP 压成一个 Conventional Commits，message 说 why。
3. **更新 [CLAUDE.md](CLAUDE.md) 末尾「当前进度」**：勾掉切片，写下次做哪个。
4. **关 session**。不要顺手再开下一个——这是上次烂尾的起点。

---

## 中断信号（出现就立即停）

| 信号 | 处理 |
|---|---|
| 同一个问题 AI 改 3 次还跑不通 | `git reset` 到上一个干净 commit，切片再拆一半 |
| 开始"修 bug 引新 bug、补丁摞补丁" | 回滚。这是 vibe 失控的典型死法 |
| AI 提议加 SPEC 没写的抽象/库 | 停。先讨论要不要更新 SPEC |
| 你解释不清刚写的代码 why | 让 AI 讲到你懂，或者删掉重来 |
| 单 session 超 90 分钟 | 承认切片太大，commit 当前可用部分，下次拆小 |

---

## 跨 session 节奏

- **一天最多 2–3 个 session**。中间留时间真的用 App、看代码、想架构。vibe 不是手不停。
- **每周一次 spec drift 检查**：花 15 分钟对照 SPEC 看代码有没有偷偷长出 Non-Goals 里的东西。drift 了就改 SPEC 或改代码，二选一，不许两边并存。
- **大决策不开 session**：换 provider / 改架构 / 加 manager → 先在文档里想清楚，更新 SPEC 后再开 session 写代码。

---

## 翻车自救（给自己留的出口）

发现这个 session 在"螺旋下坠"时（你心里那个直觉响起来时），按这个顺序救：

1. **停手**。不要再让 AI 改一次。
2. `git status` + `git diff` 看现在多脏。
3. 如果上一个 commit 是干净的 → `git reset --hard HEAD` 回去。
4. 如果上一个 commit 也是脏的 → 往前找最近一个能跑的 commit，回那里。
5. 打开 [PROJECT_SPEC.md](PROJECT_SPEC.md)，看切片描述，把它**拆成两半**。
6. 关掉这个 session。明天用更小的切片重开。

> 回滚不是失败，"明明已经螺旋下坠了还要继续救"才是失败。前两个项目就死在这一步。

---

## 焦虑时看（重要）

- **慢就是快**。前几个切片 30 分钟做完，反而是好事——说明拆得对。
- **没进度的 session 也有用**：澄清了一个边界 / 砍掉了一个不该做的功能 / 让你看清下个切片该怎么拆——都算赚到。
- **三个月做不完不丢人，烂尾才丢人**。每周一个能演示的小功能 > 一个月闷头堆一堆跑不起来的代码。
- **看不懂的代码就是别人的代码**。这个项目能不能讲清楚，比能不能跑起来更决定它是不是你的。
