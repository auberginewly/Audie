# 录音时长与内存保险丝

## 目标

取消用户可感知的固定录音时长限制。录音超过 120 秒后仍应进入现有转写、润色和注入流程，同时避免原始 PCM 缓冲无界增长。

## 约束内方案

- `AudioManager` 不再按设备采样率、声道数和时长计算 retained buffer 上限。
- 单次录音最多保留 512 MiB 的 `f32` PCM，即 134,217,728 个样本；达到上限后停止扩充缓冲并在结束录音时返回明确错误。
- 该保险丝不可由用户配置，只处理异常内存增长，不作为产品级时长限制。
- 豆包流式音频队列继续保持有界；队列溢出仍回退批量 ASR。
- `LastTake` 继续使用 `Arc<AudioData>`，并在终态 settle 后释放。

## 不做

- 不增加会议转录、长音频导入、音频落盘或分段转写。
- 不修改 command、event、manager 或 provider 接口。
- 不引入依赖或新的配置项。

## 验收

1. 48 kHz 双声道累计 125 秒样本仍低于 retained buffer 上限。
2. 未达到内存保险丝时完整保留样本，不标记溢出。
3. 单个音频 callback 跨过上限时只保留剩余容量，并标记溢出。
4. 达到上限后继续更新音量峰值，但不再增长缓冲。
5. 流式 chunk 只转发实际保留的样本。
6. 真机连续录音至少 125 秒，松手后不出现原 120 秒错误，并完成转写与光标注入。
